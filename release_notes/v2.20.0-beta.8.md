## 🪟 Dual-Axis Venetian Blind Support — Beta 8 (closes #33)

Beta.8 adds an **Inverse Tilt** option for hardware where the tilt axis is
wired in reverse. All beta.1 through beta.7 content is included.

### ✨ New in Beta 8

- **Inverse tilt option (`inverse_tilt`, #33):** A new **Inverse Tilt**
  toggle in the Venetian geometry step lets you flip the tilt axis for
  hardware that sends 0 for slats-open and 100 for slats-closed — the
  opposite of what HA expects. The integration stores the logical value
  internally and converts to the wire value only when sending the command.
  This is separate from the existing **Inverse state** option, which
  controls the position (carriage) axis. Most covers do not need this; it
  is aimed at KNX and Somfy devices where the tilt motor is wired in
  reverse relative to the HA convention.

- **Consistent tilt verification after inversion:** The readback check
  that detects tilt drift now converts the wire reading back to logical
  space before comparison, so the 5% tolerance test is correct regardless
  of inversion state. Without this, an inverted cover would report apparent
  drift of ~100% after every tilt command and retry on every cycle.

- **Clarified `max_position` help text:** The description for the Maximum
  Position option now notes that on venetian blinds it constrains the
  carriage (vertical) axis only. Slat tilt comes from sun geometry and is
  not subject to this cap.

### 🔑 Highlights (all betas)

- **New `cover_venetian` sensor type** — pick "Venetian Blind (Dual-Axis)" in
  the setup wizard. The geometry screen captures window dimensions and slat
  geometry in one step.
- **Sequenced position-then-tilt commands** — the integration sends
  `set_cover_position` first, polls `current_position` until the cover settles
  (within tolerance or no progress for three consecutive idle samples, hard-capped
  at 60 s), then sends `set_cover_tilt_position`. This eliminates the ping-pong
  fight.
- **Per-axis manual override with position-authoritative tilt suppression** —
  tilt drift inside a 90 s window after a position command is treated as motor
  back-rotate and ignored. Tilt deltas outside that window still trip manual
  override on the tilt axis.
- **Solar gate** — tilt only fires when the SOLAR handler is active, keeping
  covers quiet during overrides and non-sun-tracking conditions.
- **Venetian mode** — `tilt_only` mode skips the position command for covers
  where vertical travel is managed elsewhere.
- **Decision-trace tilt** — the Decision Trace sensor and diagnostics download
  show the tilt the engine derived alongside the resolved position.
- **Configurable upper retraction guard (beta.3)** — "Skip Tilt Above Position"
  (95% default) prevents unnecessary tilt commands when slats are retracted
  into the housing.
- **Inverse tilt (beta.8)** — hardware-level tilt inversion for KNX/Somfy
  devices wired opposite to the HA convention.

### ✨ Features

- **Dual-axis venetian (`cover_venetian`) (#33):** New sensor type that owns
  both vertical position and slat tilt on a single HA cover entity. The pure
  calculation engine (`engine/covers/venetian.py:VenetianCoverCalculation`) was
  already in place and unit-tested; this release wires it through the integration
  layer — `config_flow`, coordinator, command service, manual_override manager,
  decision trace, and diagnostics.
- **Position-then-tilt sequencing in `CoverCommandService`:** A new
  `_wait_for_position_settle()` helper polls `current_position` every 500 ms
  until the entity reports a value within `POSITION_TOLERANCE_PERCENT` of the
  target, the position stops changing for three consecutive idle samples, or the
  60 s timeout fires. `apply_position()` then dispatches
  `set_cover_tilt_position` with the engine-derived tilt.
- **Tilt-axis manual override with venetian suppression:** For `cover_venetian`
  the manager evaluates both axes per state-change event, but tilt drift inside
  the 90-second window after a position command is recorded as
  `manual_override_rejected_tilt_suppression` instead of tripping the override.
- **Skip tilt above threshold (beta.3, #33):** `VenetianPolicy.after_position_command()`
  returns early when `position > tilt_skip_above`, skipping the tilt sequence
  entirely. The threshold defaults to 95% and is configurable via
  `venetian_tilt_skip_above` in the Venetian geometry step.
- **Tilt-only mode (beta.6, #33):** `venetian_mode = tilt_only` skips the
  position command and sends only `set_cover_tilt_position` each cycle.
- **Solar gate for tilt (beta.6, #33):** Tilt commands are suppressed whenever
  the pipeline resolves to a handler other than SOLAR.
- **Continuous tilt tracking (beta.6, #33):** Tilt is recalculated and emitted
  every update cycle regardless of whether the resolved position changed.
- **Target Tilt sensor (beta.2, #33):** Diagnostic sensor exposing the engine's
  target tilt angle in degrees.
- **Inverse tilt (beta.8, #33):** `DualAxisSequencer` converts logical tilt to
  wire value via `_to_wire()` when `invert_tilt` is enabled. The logical value
  is stored in `_tilt_targets`; the wire value is sent to HA and shown in the
  log alongside the logical value for debugging.

### 🐛 Bug Fixes

- **Tilt verification broken under inversion (beta.8, #33):** The post-command
  drift check was comparing the wire reading directly against the logical target.
  With inversion enabled this produced a ~100% apparent delta on every cycle,
  clearing the target and forcing a retry every update. The fix applies
  `_to_wire()` to the wire reading before the delta comparison, so the check
  operates entirely in logical space.

- **False manual override on tilt-only path (beta.7, #33):** `_send_tilt_command`
  now stamps the back-rotate suppression window before calling
  `set_cover_tilt_position`, so both the post-settle and tilt-only paths block
  spurious `manual_override_set` events during slat settle.

- **State-aware position settle — Shelly 2PM false-settle (beta.6, #33):** The
  settle poll was counting idle samples without checking whether the cover was
  still in motion. Three consecutive equal readings triggered "settled" while
  the motor was mid-travel, firing tilt prematurely. The stall counter now only
  increments when `cover.state` is not opening or closing.

- **Tilt drift stuck on stale target (beta.6, #33):** After a tilt command, the
  actual tilt reported by the cover could differ from the commanded angle by more
  than the hardware tolerance. Without the readback check, the coordinator treated
  the stale target as satisfied and stopped retrying. The fix clears the target
  when drift exceeds `VENETIAN_TILT_VERIFY_TOLERANCE` (5%), forcing a retry on
  the next cycle.

- **Boundary drift false override (beta.5, #33):** Drift exactly equal to the
  position tolerance floor passed the strict `<` check and reached the
  manual-override handler. Changing to `<=` closes the gap.

- **Two cooperating manual override bugs (beta.4, #33):** `SecondaryAxisCheck.evaluate`
  short-circuited to `consumed=False` when `new_value == expected`, skipping the
  suppression callback and letting position-axis drift reach the manual-override
  handler. Separately, `_rebase_commanded_position` captured `current_position`
  before the motor's back-drive settled, so the rebase was skipped and the
  commanded target drifted from the actual settled position. The fix checks
  suppression before the on-target early exit and adds a 1.5-second settle delay
  before the rebase.

- **False manual override from back-rotate drift (beta.3, #33):** When the
  back-rotate settled after the 90 s suppression window, `SecondaryAxisCheck`
  was letting the event propagate unconsumed into the manual-override handler.
  Setting `consumed=True` on the suppression-rejection path closes that gap.

- **Post-tilt reconciliation loop (beta.2, #33):** After `set_cover_tilt_position`
  fired, each subsequent `current_tilt_position` change re-triggered a tilt
  command. The reconciliation loop is now in-flight-aware and skips re-sends
  while the motor is settling.
