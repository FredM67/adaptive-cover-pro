## 🪟 Dual-Axis Venetian Blind Support — Beta 4 (closes #33)

Beta.4 closes two cooperating bugs that caused spurious manual override trips during
normal position+tilt cycles on venetian blinds. Both were surfaced by beta.3 testers.
All beta.1 through beta.3 features are included.

> ### 🙏 Looking for Beta Testers
>
> Beta.4 is calling in the volunteers from [#33]: **fnep**, **x4N70pHyLL**,
> **geryyyyyy** (Somfy IO), **MetalMinotaur** (KNX), and **jkuchar** (Shelly 2PM).
> Please install beta.4 and confirm on issue #33 that:
>
> 1. No spurious "manual override" trips occur during a normal position+tilt cycle.
> 2. Tilt commands no longer fire when the blind is raised near the top.

### ✨ New in Beta 4

- **Two-bug manual override fix (#33):** `SecondaryAxisCheck.evaluate` was
  short-circuiting to `consumed=False` whenever `new_value == expected`, bypassing
  the suppression callback entirely. Simultaneously, `_rebase_commanded_position`
  read `current_position` immediately after `set_cover_tilt_position` returned,
  before the motor finished its back-drive; the pre-settle value matched the target
  so the rebase was skipped, leaving the commanded position out of sync with where
  the cover actually landed. Together the two bugs let position-axis drift trip
  manual override on covers that were behaving correctly. The fix checks suppression
  before the on-target early exit in `evaluate()` and adds a 1.5-second settle
  delay before the rebase.

### ✨ New in Beta 3

- **Skip tilt when cover is retracted (#33):** A new option — **Skip Tilt Above
  Position (%)** — guards the tilt sequence so `set_cover_tilt_position` is never
  sent when the commanded position exceeds the threshold (default: **95%**). At
  high positions the slats retract into the housing and tilting is physically
  meaningless; issuing the command anyway forces an unnecessary motor back-drive.
  The threshold is configurable per-instance in the Venetian geometry step.

- **False manual override fix during back-rotate (#33):** `SecondaryAxisCheck`
  now sets `consumed=True` while inside the tilt-suppression window. Without this,
  back-rotate drift arriving after the grace period closed was not marked as
  consumed and continued propagating to the manual-override handler, tripping
  the override incorrectly. The fix ensures that any position change the
  suppression logic rejects is fully consumed and never reaches the override path.

### ✨ New in Beta 2

- **Target Tilt sensor (#33):** A new diagnostic sensor exposes the tilt angle the
  engine is targeting (in degrees), making it easy to verify the slat-angle
  calculation without downloading a full diagnostics bundle. Visible in the HA
  entity list alongside the existing Target Position sensor.

- **Post-tilt reconciliation loop fix:** After `set_cover_tilt_position` fired,
  the coordinator was treating each subsequent `current_tilt_position` state
  change as a new deviation and re-sending the tilt command. The fix makes the
  reconciliation loop aware of in-flight tilt commands and skips re-sends while
  the motor is settling.

- **`CoverTypePolicy` refactor (internal):** Venetian-specific geometry schema,
  entity-selector domains, and glare-zone support are now gated through a
  `CoverTypePolicy` interface. This is a pure internal change with no user-visible
  behavior, but it removes several `if sensor_type == "cover_venetian"` branches
  that had accumulated in the coordinator and config flow.

### 🔑 Highlights (all betas)

- **New `cover_venetian` sensor type** — pick "Venetian Blind (Dual-Axis)" in the
  setup wizard. The geometry screen captures BOTH window dimensions and slat
  geometry in one step.
- **Sequenced position-then-tilt commands** — when the engine resolves a new
  position, the integration sends `set_cover_position` first, polls
  `current_position` until the cover settles (within tolerance, or no progress for
  three consecutive samples, hard-capped at 60 s), and only then sends
  `set_cover_tilt_position`. This eliminates the ping-pong fight.
- **Per-axis manual override with position-authoritative tilt suppression** —
  tilt drift inside a 90 s window after a position command is treated as motor
  back-rotate and ignored. Tilt deltas outside that window (e.g. user grabs the
  wand) still trip manual override on the tilt axis. Position-axis manual
  override behaves exactly as it does for `cover_blind`.
- **Decision-trace tilt** — the Decision Trace sensor and diagnostics download
  now show the tilt the engine derived alongside the resolved position.
- **Configurable retraction guard (beta.3)** — the "Skip Tilt Above Position"
  threshold prevents unnecessary tilt motor commands when the blind is raised.

### ✨ Features

- **Dual-axis venetian (`cover_venetian`) (#33):** New sensor type that owns
  both vertical position AND slat tilt on a single HA cover entity. The pure
  calculation engine (`engine/covers/venetian.py:VenetianCoverCalculation`) was
  already in place and unit-tested; this beta wires it through the integration
  layer — `config_flow`, coordinator, command service, manual_override manager,
  decision trace, and diagnostics.
- **Position-then-tilt sequencing in `CoverCommandService`:** A new
  `_wait_for_position_settle()` helper polls `current_position` every 500 ms
  until the entity reports a value within `POSITION_TOLERANCE_PERCENT` of the
  target OR the position stops changing for three consecutive samples OR the 60 s
  timeout fires. `apply_position()` then dispatches `set_cover_tilt_position`
  with the engine-derived tilt.
- **Tilt-axis manual override with venetian suppression:** For `cover_venetian`
  the manager evaluates BOTH axes per state-change event, but tilt drift inside
  the 90-second window after a position command is recorded as
  `manual_override_rejected_tilt_suppression` instead of tripping the override.
- **Skip tilt above threshold (beta.3, #33):** `VenetianPolicy.after_position_command()`
  returns early when `position > tilt_skip_above`, skipping the tilt sequence
  entirely. The threshold defaults to 95% and is configurable via the new
  `venetian_tilt_skip_above` option in the Venetian geometry step.
- **Target Tilt sensor (beta.2, #33):** Diagnostic sensor exposing the engine's
  target tilt angle in degrees alongside the existing Target Position sensor.

### 🐛 Bug Fixes

- **Two cooperating manual override bugs (beta.4, #33):** `SecondaryAxisCheck.evaluate`
  short-circuited to `consumed=False` when `new_value == expected`, skipping the
  suppression callback and letting position-axis drift reach the manual-override
  handler. Separately, `_rebase_commanded_position` captured `current_position`
  before the motor's back-drive settled, so the rebase was skipped and the
  commanded target drifted from the actual settled position. Covers behaving
  normally were tripping manual override as a result. The fix checks suppression
  before the on-target early exit and adds a 1.5-second settle delay before the
  rebase.

- **False manual override from back-rotate drift (beta.3, #33):** When the
  back-rotate settled after the 90 s suppression window, `SecondaryAxisCheck`
  was letting the event propagate unconsumed into the manual-override handler.
  Setting `consumed=True` on the suppression-rejection path closes that gap and
  prevents spurious override trips.

- **Post-tilt reconciliation loop (beta.2, #33):** After `set_cover_tilt_position`
  fired, each subsequent `current_tilt_position` change re-triggered a tilt
  command, creating an endless send loop. The reconciliation loop is now
  in-flight-aware and skips re-sends while the motor is settling.
