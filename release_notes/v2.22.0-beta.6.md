# v2.22.0-beta.6

beta.6 adds two companion entities that expose My Position on the managed cover, completing a feature requested in [#409]. The recall path is pipeline-aware: `force_override` (100), `weather` (90), and custom-position slots configured above priority 80 can preempt the move.

## 🎯 Should I install this?

- **If you use physical motor remotes with a saved "My" preset and want automations or dashboards to trigger that position through the managed entity**, install this. Two new entities appear per managed cover: a button to recall the preset and a number to configure or adjust the value at runtime.
- **If you are on beta.5 and it was working**, the update is low-risk — no existing behavior changes. The two new entities appear automatically once a cover entity is configured; existing automations are unaffected.
- **If you are on v2.21.5 without any active issue**, stay there. This beta exists to close out feature work before the v2.22.0 stable release.

## 🔑 What's new in beta.6

### ✨ Features

- **Pipeline-aware My Position button and number ([#409], PR [#410], commit ff6388c):**

  Two new entities appear on each configured managed cover:

  **`button.<managed_cover>_my_position` ("My Position"):** Pressing the button reads `CONF_MY_POSITION_VALUE` from config options and calls `coordinator.async_apply_user_position` with `trigger="my_position_recall"` and `force=False`. The pipeline preemption path runs first: if a higher-priority handler is active — `force_override` (priority 100), `weather` (90), or any custom-position slot configured above `ManualOverrideHandler` priority — the move is dropped and recorded as a preempted skip rather than silently ignored. When the move proceeds, min-mode floor clamping from active custom-position sensors applies automatically and `mark_user_command` engages manual override, so the cover holds the recalled position until the next coordinator cycle releases it.

  **`number.<managed_cover>_my_position_value` ("My Position Value", `EntityCategory.CONFIG`):** A runtime slider (1–99 %, step 1) that exposes `CONF_MY_POSITION_VALUE`. Writing a new value goes through `validate_options_patch` and `apply_options_patch` — the same path used by the options service — so the `sunset_use_my ⇒ my_position_value` cross-field validation still fires. The entity's `_attr_native_min_value` and `_attr_native_max_value` are pulled from `_RANGE_MY_POSITION = (1, 99)` in `const.py`.

  Both entities are registered under the `number` platform (new `Platform.NUMBER` entry in `PLATFORMS`) and carry translation keys `"my_position"` (button) and `"my_position_value"` (number) in `en.json`, `de.json`, and `fr.json`.

  The implementation is integration-agnostic — the routing layer selects `set_cover_position` for position-capable covers and falls back to the Somfy stop-while-stationary trick automatically.

## 📖 Cumulative scope (v2.22.0 series)

beta.6 is cumulative. Everything in [v2.22.0-beta.1] through [v2.22.0-beta.5] is included:

- **Nine-phase internal refactor (Phases A–H, beta.1/beta.2):** Coordinator responsibility distributed into `managers/`, `state/`, `pipeline/`, and `cover_types/`. Coordinator is ~640 lines lighter vs v2.21.5. Manual-override detection in `managers/cover_command/state_classifier.py`; cover-type-specific behaviour on `CoverTypePolicy` subclasses; cover-type literal branching eliminated. Net diff for the refactor: 62 files, +3,682 / −1,251 lines.

- **Post-settle cap grace — false venetian override fix ([#33], beta.2):** `DualAxisSequencer.is_in_suppression_with_cap` holds the cap bypass active for 5 seconds after a position command stamp, preventing KNX, Shelly 2PM, FGR223, and similar actuators from tripping false manual-override when `cover.state` settles before the tilt-walk burst finishes publishing.

- **Phase I — config-flow plumbing refactor (beta.3):** Collapsed duplicated schema reads, step routing, and options-step boilerplate into shared helpers, reducing `config_flow.py` by 168 lines.

- **Legacy climate step + optional-key cleanup (beta.3):** Removed `async_step_climate` and `CLIMATE_SCHEMA` dead code; extracted optional-key constants for `WEATHER_OVERRIDE`, `LIGHT_CLOUD`, and `TEMPERATURE_CLIMATE` schemas to prevent [#377]-class drift; fixed a latent omission where `CONF_CLOUDY_POSITION` was missing from its inline optional-key list.

- **"Is Sunny" sensor can now be cleared in the options flow ([#377], beta.3):** `CONF_BINARY_SENSOR` added to the optional-entities list so a blank submission removes the prior value instead of silently keeping it.

- **MODE2 tilt climate handler: sun-side awareness + `min_position` warning ([#373], beta.4):** The climate handler now picks the correct MODE2 hemisphere from the sign of gamma; `TiltPolicy.climate_tilt_percentage` is the single conversion helper for all four call sites. A ⚠️ warning surfaces in the config-flow summary when MODE2 is active and `min_position` ≥ 50.

- **Venetian tilt-axis command-grace suppression ([#33], beta.4):** `VenetianPolicy.secondary_axis_check` now carries a suppression callback that OR-composes the back-rotate window with a `_is_in_tilt_command_grace` helper, closing the last false-override vector on the tilt axis during a command-grace window.

- **Venetian publish-lag anchored to settle transition + startup grace ([#33], beta.5):** `VENETIAN_POSITION_SETTLE_STARTUP_GRACE_SECONDS = 6.0` blocks premature stall declaration on slow-start actuators (Somfy IO, 3–5 s motor startup delay); `VENETIAN_BACKROTATE_PUBLISH_LAG_SECONDS = 45.0` anchors the cap bypass to the observed `moving → settled` transition rather than to `stamp_position_command`, closing the last false-override vector on slow-bus republish paths.

- **Pipeline-aware My Position button and number ([#409], beta.6, above).**
