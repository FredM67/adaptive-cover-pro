# v2.21.1

## 🔑 Highlights

Two regressions from v2.21.0 ship as a fast-follow patch. The venetian dual-axis sequencer had four interacting bugs that caused the blind to loop — a user opening the blind during the back-rotate window was silently absorbed as the new commanded target, triggering a re-close cycle. The `adaptive_cover_pro.set_position` service shipped fully working in v2.21.0 but was never declared in `services.yaml`, so HA's UI had no service picker, no field selector, and no description text for it. Both fixes are isolated; no config migration or reload is required.

## 🐛 Bug Fixes

- **Venetian dual-axis sequencer loop** (#33, PR #376) — After a full position-and-tilt cycle, the sequencer opened a 90-second back-rotate suppression window to absorb motor-induced slat drift. Four interacting bugs let a user opening the blind during that window get silently adopted as the new commanded target, triggering a re-close cycle that appeared as a loop. The fixes address each failure mode separately: post-tilt position rebase now requires the settle to have succeeded (no rebase on timeout or stall); drift above `VENETIAN_REBASE_MAX_DRIFT_PERCENT` (15%) is refused and logged rather than silently adopted; tilt deltas above `VENETIAN_BACKROTATE_MAX_DELTA_PERCENT` (30%) fall through to manual-override even inside the suppression window, since a delta that large is a user action rather than motor back-drive; and the suppression stamp moves from `_send_tilt_command` to `run_sequence` so that tilt-only sends from `update_tilt_only` no longer re-extend the window on every solar recompute. The default `DEFAULT_VENETIAN_POST_SETTLE_HOLD_SECONDS` is also bumped from 2.0 to 3.0 seconds, reflecting tester feedback that 3 s is more reliable across FGR223 and similar actuators.

- **`set_position` service missing from `services.yaml`** (#372, PR #375) — The `adaptive_cover_pro.set_position` service introduced in v2.21.0 was registered in Python but had no entry in `services.yaml`. Without that descriptor, HA's Developer Tools service picker showed no target selector, no `position` field, and no description text; dashboard and script authors who tried to call the service by name had no UI surface to build from. The fix adds the full YAML descriptor — including target entity selector, slider field (0–100%), and description — matching the pattern used by sibling services. A new structural test guards against the same omission for any service registered in future.
