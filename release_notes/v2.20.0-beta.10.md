## 🪟 Dual-Axis Venetian Blind Support — Beta 10 (closes #33)

Beta.10 fixes a slat-reassertion bug on fully-retracted venetian blinds,
adds a symmetric **Minimum Slat Tilt** floor, and ships two manual-override
detection fixes: assumed-state covers and slow-reporting Zigbee covers
that misfire mid-traverse.

### ✨ New in Beta 10

- **Retract-tilt overwrite (fix, #33 comment #54):** When the venetian
  carriage is commanded above `tilt_skip_above` (default 95%, e.g. when the
  sun leaves the FOV and the engine retracts to position=100), the
  integration now sequences a neutral tilt command (`POSITION_OPEN`) after
  the carriage settles. KNX and Shelly actuators retain their last
  commanded tilt internally and reapply it ~1-2 seconds after the carriage
  settle — without overwriting that cache, the prior solar-cycle tilt
  reasserts and closes the slats on a fully-retracted blind. The fix
  rewrites `VenetianPolicy.after_position_command` to fire `run_sequence`
  with `tilt_target=POSITION_OPEN` instead of returning early.

- **Minimum Slat Tilt (`min_tilt`, #33 comment #55):** A new **Minimum
  Slat Tilt (%)** field in the Venetian geometry step (0–100, default 0)
  floors the tilt angle the engine will ever command. Useful for blinds
  that block all light at 0% — set to e.g. 10% to keep some light through
  at all times. Symmetric to the beta.9 `max_tilt` cap; the engine clamps
  the computed tilt to `[min_tilt, max_tilt]` via the new
  `VenetianCoverCalculation._clamp_tilt` helper. The floor also applies to
  the NaN-fallback path, so geometry-undefined cycles respect the user's
  minimum.

- **Assumed-state manual override (fix):** Covers exposed via integrations
  that report `assumed_state=true` (some KNX, MQTT, and template covers)
  don't drive real state-change events when HA dispatches a command. The
  manual-override manager now consults the HA `Context` on state-change
  events to distinguish commands issued by Adaptive Cover Pro from
  commands issued elsewhere, so manual override no longer trips on the
  integration's own writes against assumed-state entities.

- **Slow Zigbee cover false manual override (fix, #361):** Some Zigbee
  covers (e.g. ZHA Rollershade) only report `current_position` at the end
  of motion. When motion took longer than the 45 s transit timeout, the
  first state-change event arrived mid-traverse with `state=closing` and
  a stale position; the position-math check then misread the gap as a
  manual nudge against the integration's own command. The manual-override
  manager now skips the position-math path when the cover's own state
  attribute reports `opening`/`closing` at event time, deferring the
  check to the settling event with the true final position. Tilt-axis
  manual changes during a position traverse are unaffected.

### 🐛 Bug Fixes

- **Slats close on fully-retracted blind (beta.10, #33 comment #54):**
  Above `tilt_skip_above`, `VenetianPolicy.after_position_command` no
  longer returns early. It sequences a tilt to `POSITION_OPEN` so the
  actuator's tilt cache is overwritten with a known neutral value.

- **False manual override on assumed-state covers (beta.10):**
  `ManualOverrideManager._on_state_change` now uses the HA `Context` of
  the state-change event to detect commands issued by the integration
  itself, replacing the previous time-based-only check that failed for
  covers without real motor state.

- **False manual override on slow Zigbee covers (beta.10, #361):**
  `handle_state_change` now records
  `manual_override_rejected_in_transit` and returns early when the
  cover's state attribute is `opening`/`closing` at event time, so
  mid-traverse position reports from end-of-motion-only Zigbee covers
  no longer count as user nudges. The settling event still runs the
  full check, catching real manual moves one event later.
