## 🪟 Dual-Axis Venetian Blind Support (closes #33)

This is the first beta of Adaptive Cover Pro's `cover_venetian` sensor type — a
single instance that drives BOTH `set_cover_position` AND `set_cover_tilt_position`
on the same Home Assistant entity. Until now, users on KNX, Somfy IO/Tahoma,
Shelly 2PM, and similar dual-axis hardware had to run two paired instances (one
`cover_blind`, one `cover_tilt`) against the same physical blind. That arrangement
caused a recurring pain reported in [#33]: every time the vertical instance moved
the cover, the slats back-rotated as a side-effect of the motor's mechanics, and
the tilt instance read that drift as a manual override and stopped automating.
The two instances also fought each other on a 5-minute cycle. This release ships
a dedicated dual-axis cover type that owns both axes from one config entry.

### 🔑 Highlights

- **New `cover_venetian` sensor type** — pick "Venetian Blind (Dual-Axis)" in the
  setup wizard. The geometry screen captures BOTH window dimensions and slat
  geometry in one step.
- **Sequenced position-then-tilt commands** — when the engine resolves a new
  position, the integration sends `set_cover_position` first, polls
  `current_position` until the cover settles (within tolerance, or no progress for
  three consecutive samples, hard-capped at 60 s), and only then sends
  `set_cover_tilt_position`. This eliminates the ping-pong fight.
- **Per-axis manual override with position-authoritative tilt suppression** —
  tilt drift inside a 90 s window after a position command is treated as motor
  back-rotate and ignored. Tilt deltas outside that window (e.g. user grabs the
  wand) still trip manual override on the tilt axis. Position-axis manual
  override behaves exactly as it does for `cover_blind`.
- **Decision-trace tilt** — the Decision Trace sensor and diagnostics download
  now show the tilt the engine derived alongside the resolved position. A new
  synthetic terminal step labelled `venetian_engine` records the slat-angle
  computation.
- **Translations** — DE and FR strings shipped for the new cover type label.

### ✨ Features

- **Dual-axis venetian (`cover_venetian`) (#33):** New sensor type that owns
  both vertical position AND slat tilt on a single HA cover entity. The pure
  calculation engine (`engine/covers/venetian.py:VenetianCoverCalculation`) was
  already in place and unit-tested; this beta wires it through the integration
  layer — `config_flow`, coordinator, command service, manual_override manager,
  decision trace, and diagnostics.
- **Position-then-tilt sequencing in `CoverCommandService`:** A new
  `_wait_for_position_settle()` helper polls `current_position` every 500 ms
  until the entity reports a value within `POSITION_TOLERANCE_PERCENT` of the
  target OR the position stops changing for three consecutive samples (handles
  hardware that overshoots and back-tracks) OR the 60 s timeout fires.
  `apply_position()` then dispatches `set_cover_tilt_position` with the
  engine-derived tilt. Real motors that back-rotate the slats during vertical
  motion finish their full sequence before our tilt command lands, so the two
  commands no longer fight.
- **Tilt-axis manual override with venetian suppression:**
  `AdaptiveCoverManager.handle_state_change()` accepts new optional `our_tilt`
  and `is_in_tilt_suppression` parameters. For `cover_venetian` the manager
  evaluates BOTH axes per state-change event, but tilt drift inside the
  90-second `VENETIAN_TILT_SUPPRESSION_SECONDS` window after a position command
  is recorded as `manual_override_rejected_tilt_suppression` instead of
  tripping the override. Position-only and non-venetian behavior is unchanged.

### 📖 Documentation

- New wiki page **[Venetian Blinds]** covering geometry inputs, sequenced-command
  timing, supported integrations confirmed by testers (KNX, Somfy IO, Shelly 2PM),
  and the migration path for users currently running paired `cover_blind` +
  `cover_tilt` instances against the same entity.
- New `Venetian Blind (Dual-Axis)` entry in the cover-type selector. Existing
  `cover_blind`, `cover_awning`, and `cover_tilt` instances are not affected.

### 🚦 Out of scope for v2.20.0 (planned for follow-ups)

- **Auto-suspend on physical switch open/close** (x4N70pHyLL's request from #33):
  the pattern of "fully open via physical switch suspends auto-control until
  fully closed" is intentionally deferred — it raises questions that span all
  cover types, not just venetian, and warrants its own design pass. Tracked in
  `todo.md` under "Issue #33 Venetian Follow-ups". Will land in a later release.
- **Per-axis override config** (force_override_tilt, custom_position_N_tilt,
  etc.) — for v2.20.0, scalar overrides set position only; tilt is engine-derived.
  Adding optional companion tilt fields per override is a follow-up.
- **Cloud-suppression staged tilt (#175)** — now unblocked by dual-axis
  venetian. Pick up separately.
- **Glare zones for venetian** — currently `cover_blind` only. Add later if
  requested.
- **Guided merge flow** for users with paired `cover_blind` + `cover_tilt`
  instances — beta.1 ships doc-only migration via the new wiki page.
