## 🎯 Hold Position on Vacancy, Config Flow UX Overhaul, and Bug Fixes

### 🔑 Highlights

- New `hold_position` motion timeout mode keeps covers at their current position when occupancy clears, instead of returning to default — useful for rooms where you want manual choices to stick until someone re-enters (#335).
- Climate handler now correctly inverts positions for horizontal awnings: 100% is extended (sun blocked), 0% is retracted (sun in). Previously, awnings received blind semantics and moved in the wrong direction in climate mode (#337).
- Every config and options screen now shows a contextual description and a wiki link — the instance name appears at the top of the options menu so you always know which cover you're editing.
- Custom-position binary sensor edge triggers now correctly bypass the time-delta gate. Previously, a sensor toggle within `delta_time` of the last movement was silently dropped (#348).
- Entity selectors for on/off, numeric, and motion fields now accept a wider range of domains — `device_tracker`, `person`, `zone`, `switch`, `schedule`, `input_number` — reducing friction for non-standard setups (#319).

### ✨ Features

- **Hold position on vacancy (`hold_position` motion timeout mode) (#335):**
  A new `motion_timeout_mode` option in the Motion Override config step controls what
  happens when the motion timeout expires. `return_to_default` (the existing behavior)
  sends the cover to its default position. `hold_position` freezes the cover where it
  is — no move command is issued. When occupancy is re-detected, control resumes and
  the cover moves to the currently calculated position. The hold exits naturally when
  the active time window closes or the sun leaves the window's field of view. A new
  `"holding"` state on the `motion_status` sensor shows when hold is active, and
  `motion_hold_active` is included in the diagnostics download.

- **Widen entity selector domains for on/off, numeric, and motion inputs (#319):**
  Config flow selectors for on/off fields (force override, weather, custom position,
  and motion) previously accepted only `binary_sensor` and `input_boolean`. They now
  also accept `device_tracker`, `person`, `zone`, `switch`, and `schedule`. Numeric
  fields (lux, irradiance, temperature) now accept `sensor` and `input_number` across
  the board. Motion detection uses a shared `is_entity_active()` helper that evaluates
  all entity types consistently across the pipeline.

- **Instance name in options flow menu:**
  The options flow main menu now displays the config entry title (cover name) as a
  subtitle so you can confirm which cover you're editing when multiple instances are
  configured.

- **Contextual descriptions and wiki links on every config/options screen:**
  Every step in both the initial setup wizard and the options flow now includes a
  description of the screen's purpose and its key fields. Each screen carries a
  {learn_more} link that resolves to the correct wiki page at runtime — type-specific
  pages (Vertical / Horizontal / Tilt) for the Geometry screen. All descriptions are
  translated into DE and FR.

### 🐛 Bug Fixes

- **Climate handler inverts positions for horizontal awnings (#337):**
  The climate pipeline handler used blind semantics for all cover types: `100%` meant
  "raised (sun in)" and `0%` meant "lowered (sun blocked)". For horizontal awnings the
  semantics are opposite — `100%` is extended (sun blocked) and `0%` is retracted (sun
  in). In winter mode, awnings were being sent to `0%` when the intent was to let sun
  in, and to `100%` when the intent was to block sun. A new `_position_for_intent()`
  helper maps `sun_through` / `sun_blocked` intent to the correct scalar per cover
  type. Blind and tilt cover behavior is unchanged.

- **Suppress cover commands for unloaded or unavailable entities (#342):**
  On HA restart, some platforms (Homematic IP is the canonical case) register cover
  entities late. The integration's first refresh would call `set_cover_position` before
  the entity existed, producing a "missing or not currently available" warning and, on
  platforms that queue commands, replaying the wrong target once the entity came online.
  `CoverCommandService.apply_position` now short-circuits when `hass.states.get(entity_id)`
  returns `None` or the state is `unavailable`. The coordinator requests a refresh when
  a tracked cover transitions from "not registered" to a real state, so the
  originally-skipped position is recomputed and dispatched once the entity is reachable.

- **Custom-position sensor edge triggers bypass the time-delta gate (#348):**
  When a custom-position binary sensor toggled within `delta_time` minutes of the last
  cover movement, the resulting command was silently dropped. The same-position
  short-circuit added in v2.18.4 (#300) suppresses redundant re-sends when the cover
  is already at the target but did not protect edge-triggered sensor changes. The fix
  records the triggering entity in `async_check_entity_state_change` and passes
  `force=True` to `apply_position` when that entity matches a configured custom-position
  sensor. Solar-driven refreshes with an active sensor keep `force=False`, preserving
  the v2.18.4 invariant.

- **Sill shadow regression from v2.19.0-beta.4 corrected (#358):**
  A geometry fix introduced in beta.4 incorrectly returned `h_win` (fully open) when
  the sill shadow depth exceeded the shaded distance. The geometrically correct answer
  is `position=0` (fully closed) — when the sill alone intercepts all sun penetration,
  every ray at the shaded boundary is still above the floor and no blind movement helps.
  v2.19.0 stable restores the pre-beta.4 clamp (`effective_distance = max(effective_distance, 0)`).
  If you installed a beta and set `inverse_state=True` in response to the now-corrected
  beta.4 release notes advice, revert that change after upgrading.

- **Custom position slot fields not cleared when user empties them (#323):**
  Clearing a slot's sensor, target position, or priority in the UI silently kept the
  previous value. A shared `_CUSTOM_POSITION_OPTIONAL_KEYS` list now ensures both the
  initial setup and options flows zero out omitted fields.

- **Transit timeout dropped when syncing Manual Override settings:**
  `CONF_TRANSIT_TIMEOUT` was absent from the `manual_override` frozenset in
  `SYNC_CATEGORIES`. Copying Manual Override settings to other covers silently omitted
  the transit timeout. It is now included.

- **Last action sensor showed UTC time instead of local time:**
  The last cover action timestamp was formatted from a UTC-aware datetime without
  converting to the HA local timezone first, so users in non-UTC zones saw the wrong
  time. The sensor now converts to local time before formatting.

### 🧹 Chores

- **Three-tier internal cleanup (#320 / #321 / #322):** Typo fixes, dead-code removal,
  and type-hint tightening (tier 1); shared `apply_minimum_mode()` helper extracted
  from three near-identical inline blocks across the force, weather, and custom-position
  handlers (tier 2); four custom-position slot key names centralized in a single
  `CUSTOM_POSITION_SLOTS` map in `const.py` so coordinator, config flow, and sensor
  enablement all iterate the same source (tier 3). No runtime behavior changes.

- **`PerEntityState` dataclass in `CoverCommandService`:** Per-entity mutable state
  (last command time, last position, retry counters) is now grouped in a single
  `PerEntityState` dataclass instead of spread across three parallel dicts.

- **Dict-driven entity spec/factory for sensor, switch, and binary_sensor platforms:**
  Entity construction is now driven by a central spec dictionary instead of inline
  conditionals, making it easier to add new entity types without touching multiple
  call sites.

- **Dependency bumps:** pvlib updated from `~=0.11` to `~=0.15` (#331); testing,
  notebook, and colorlog dev dependencies bumped to current versions (#328–#330).
