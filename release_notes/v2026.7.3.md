## Highlights

2026.7.3 is a bug-fix patch for the July line. It stops the sunset/end-time dispatch from overwriting an active custom-position slot mid-transition, gets the config summary actually rendering in German and French by resolving a language Home Assistant never supplies, and closes a false-override window on venetian Tilt Drift Reset when a slow actuator's endpoint publish lands after the command grace expires. No configuration changes and nothing to migrate.

## Fixed

- **Sunset/end-time dispatch no longer overwrites an active custom position (#895):** The astronomical-sunset transition and the end-of-window default dispatch both bypassed the pipeline registry and force-sent the raw sunset/default position, so a CUSTOM_POSITION slot holding a user's sleep-mode floor got overwritten mid-transition and only self-corrected on the next refresh cycle — a spurious double-move. A new `_pipeline_has_active_override()` predicate on the coordinator reports whether a non-DEFAULT handler currently owns the pipeline result; both `_on_window_closed` and `WindowTransitionTracker.check_sunset_window` (which now takes a `has_active_override` argument) consult it and skip their dispatch when a higher-priority handler is winning. The existing per-cover `is_cover_manual` check is untouched — it covers MANUAL, this covers the rest of the priority chain.

- **Config summary now renders in German and French (#905):** Home Assistant never populates `context["language"]` for config/options flows, so the narrative config-summary block always fell through to English and the `summary_i18n/{de,fr}.json` bundles from #258 sat unused. A new `_resolve_summary_language()` helper resolves the summary's language as `context.get("language") or hass.config.language or "en"`, so it now follows the HA instance's configured language while still preferring a per-user flow language if HA ever starts supplying one. The unrelated #227 fix — system language must never leak into the flow's menu/step labels — still holds; the read is scoped to this one summary call.

- **Venetian Tilt Drift Reset no longer fires a false manual override on slow actuators (#927):** A drift reset drives the slats to a mechanical endpoint and back; on slow Somfy IO/Overkiz actuators the endpoint's `current_tilt_position` publish can land 6-7 seconds later, after the 5-second command grace has closed, and `SecondaryAxisCheck` read the stale publish as a large-delta manual move. `_maybe_drift_reset` now records a one-shot `_ResetExcursion` stamp — per entity, value-matched, time-boxed to the configured publish-lag window — whenever `_send_tilt_command` (which now reports back whether it actually dispatched) confirms the endpoint command went out; a dry-run or failed send leaves no stamp behind. `is_reset_excursion_publish()` matches the late publish against that stamp and is wired onto the tilt axis only, via a new `SecondaryAxisCheck.excursion_match` callback checked before the grace gate — and a new `consume_excursion()` pops a matching stamp even when the manager's own grace window rejects the update first, so a fast actuator's publish can't leave a stamp lingering to swallow a later genuine move. A user move to any other value still trips the override.
