## 🪟 Dual-Axis Venetian Blind Support — Beta 6 (closes #33)

Beta.6 adds tilt-only mode, a solar gate that prevents tilt from firing
outside sun-tracking windows, continuous tilt tracking, and two reliability
fixes. It also removes `tilt_skip_below`, which was introduced in beta.5:
a threshold that stopped tilt near the bottom rail solved a symptom rather
than the root cause. The solar gate addresses that case more directly — tilt
simply does not fire when the pipeline isn't in solar mode.

All beta.1 through beta.5 content is included.

### ✨ New in Beta 6

- **Tilt-only mode (`venetian_mode` option, #33):** A new **Venetian Mode**
  option — `position_and_tilt` (default) or `tilt_only` — lets you skip the
  position command entirely and only send `set_cover_tilt_position`. Useful
  when vertical position is held fixed or managed by a separate automation,
  or when covers like the Shelly 2PM are being driven tilt-first.

- **Solar gate for tilt (#33):** Tilt commands now only fire when the
  pipeline resolves to the SOLAR handler. Overrides, manual holds, weather
  suppression, and other pipeline outcomes suppress tilt entirely. This
  replaces the `tilt_skip_below` threshold from beta.5: rather than blocking
  tilt below an arbitrary position boundary, the gate prevents tilt whenever
  the integration is not actively sun-tracking.

- **Continuous tilt tracking (#33):** Tilt is now recalculated and sent on
  every update cycle, not only when the resolved position changes. Previously,
  a cloud passing overhead could update the position without moving it, and
  tilt would silently go stale. Now tilt tracks the sun angle independently.

- **State-aware position settle — Shelly 2PM fix (#33):** The settle poll
  now checks `cover.state` (opening/closing) before counting a no-change
  sample toward the stall limit. Shelly 2PM covers publish position at
  roughly 1-second intervals; three equal readings in a row had been read as
  "settled" even while the motor was still mid-travel, firing tilt too early.
  The stall counter now only increments when the cover reports itself as idle.

- **Tilt drift detection and recovery (#33):** After `set_cover_tilt_position`
  is sent, the actual `current_tilt_position` is read back. If the reported
  tilt drifted more than 5% from the commanded angle
  (`VENETIAN_TILT_VERIFY_TOLERANCE`), the recorded target is cleared so the
  next update cycle retries the tilt command rather than treating the stale
  target as already satisfied.

- **Tilt diagnostic events (#33):** All tilt command outcomes now emit
  EventBuffer events: `tilt_command_sent`, `tilt_command_skipped`,
  `tilt_command_verified`, `tilt_command_drift`. The diagnostics event timeline
  now covers both axes, matching the richness of existing position diagnostics.

- **Removes `tilt_skip_below` (beta.5 reversal, #33):** The lower-bound tilt
  guard introduced in beta.5 has been removed. If you configured
  `venetian_tilt_skip_below` in beta.5, that setting is ignored after
  upgrading. The solar gate handles the underlying case more directly.

### ✨ New in Beta 5

- **Skip tilt when cover is at or below lower-bound threshold (#33):**
  ~~Removed in beta.6.~~ This guard has been replaced by the solar gate —
  see above.

- **Boundary drift false override fix (#33):** The position check used a
  strict `<` comparison, so drift exactly equal to the effective tolerance
  floor fell outside the absorbed range and incorrectly tripped manual
  override after a venetian tilt back-drive. The comparison is now `<=`,
  which matches the documented "up to `POSITION_TOLERANCE_PERCENT`" semantics.

### ✨ New in Beta 4

- **Two-bug manual override fix (#33):** `SecondaryAxisCheck.evaluate` was
  short-circuiting to `consumed=False` whenever `new_value == expected`,
  bypassing the suppression callback entirely. Simultaneously,
  `_rebase_commanded_position` read `current_position` immediately after
  `set_cover_tilt_position` returned, before the motor finished its back-drive;
  the pre-settle value matched the target so the rebase was skipped, leaving
  the commanded position out of sync with where the cover actually landed.
  Together the two bugs let position-axis drift trip manual override on covers
  that were behaving correctly. The fix checks suppression before the on-target
  early exit in `evaluate()` and adds a 1.5-second settle delay before the
  rebase.

### ✨ New in Beta 3

- **Skip tilt when cover is retracted (#33):** A new option — **Skip Tilt
  Above Position (%)** — guards the tilt sequence so `set_cover_tilt_position`
  is never sent when the commanded position exceeds the threshold (default:
  **95%**). At high positions the slats retract into the housing and tilting
  is physically meaningless; issuing the command anyway forces an unnecessary
  motor back-drive. The threshold is configurable per-instance in the
  Venetian geometry step.

- **False manual override fix during back-rotate (#33):** `SecondaryAxisCheck`
  now sets `consumed=True` while inside the tilt-suppression window. Without
  this, back-rotate drift arriving after the grace period closed was not marked
  as consumed and continued propagating to the manual-override handler.

### ✨ New in Beta 2

- **Target Tilt sensor (#33):** A diagnostic sensor exposing the tilt angle
  the engine is targeting (in degrees), visible in the HA entity list alongside
  the existing Target Position sensor.

- **Post-tilt reconciliation loop fix:** After `set_cover_tilt_position` fired,
  the coordinator was treating each subsequent `current_tilt_position` state
  change as a new deviation and re-sending the tilt command. The fix makes the
  reconciliation loop aware of in-flight tilt commands.

- **`CoverTypePolicy` refactor (internal):** Venetian-specific geometry schema,
  entity-selector domains, and glare-zone support are now gated through a
  `CoverTypePolicy` interface. This is a pure internal change.

### 🔑 Highlights (all betas)

- **New `cover_venetian` sensor type** — pick "Venetian Blind (Dual-Axis)" in
  the setup wizard. The geometry screen captures window dimensions and slat
  geometry in one step.
- **Sequenced position-then-tilt commands** — the integration sends
  `set_cover_position` first, polls `current_position` until the cover settles
  (within tolerance or no progress for three consecutive idle samples, hard-capped
  at 60 s), then sends `set_cover_tilt_position`. This eliminates the ping-pong
  fight.
- **Per-axis manual override with position-authoritative tilt suppression** —
  tilt drift inside a 90 s window after a position command is treated as motor
  back-rotate and ignored. Tilt deltas outside that window still trip manual
  override on the tilt axis.
- **Solar gate** — tilt only fires when the SOLAR handler is active, keeping
  covers quiet during overrides and non-sun-tracking conditions.
- **Venetian mode** — `tilt_only` mode skips the position command for covers
  where vertical travel is managed elsewhere.
- **Decision-trace tilt** — the Decision Trace sensor and diagnostics download
  show the tilt the engine derived alongside the resolved position.
- **Configurable upper retraction guard (beta.3)** — "Skip Tilt Above Position"
  (95% default) prevents unnecessary tilt commands when slats are retracted
  into the housing.

### ✨ Features

- **Dual-axis venetian (`cover_venetian`) (#33):** New sensor type that owns
  both vertical position and slat tilt on a single HA cover entity. The pure
  calculation engine (`engine/covers/venetian.py:VenetianCoverCalculation`) was
  already in place and unit-tested; this release wires it through the integration
  layer — `config_flow`, coordinator, command service, manual_override manager,
  decision trace, and diagnostics.
- **Position-then-tilt sequencing in `CoverCommandService`:** A new
  `_wait_for_position_settle()` helper polls `current_position` every 500 ms
  until the entity reports a value within `POSITION_TOLERANCE_PERCENT` of the
  target, the position stops changing for three consecutive idle samples, or the
  60 s timeout fires. `apply_position()` then dispatches
  `set_cover_tilt_position` with the engine-derived tilt.
- **Tilt-axis manual override with venetian suppression:** For `cover_venetian`
  the manager evaluates both axes per state-change event, but tilt drift inside
  the 90-second window after a position command is recorded as
  `manual_override_rejected_tilt_suppression` instead of tripping the override.
- **Skip tilt above threshold (beta.3, #33):** `VenetianPolicy.after_position_command()`
  returns early when `position > tilt_skip_above`, skipping the tilt sequence
  entirely. The threshold defaults to 95% and is configurable via
  `venetian_tilt_skip_above` in the Venetian geometry step.
- **Tilt-only mode (beta.6, #33):** `venetian_mode = tilt_only` skips the
  position command and sends only `set_cover_tilt_position` each cycle.
- **Solar gate for tilt (beta.6, #33):** Tilt commands are suppressed whenever
  the pipeline resolves to a handler other than SOLAR.
- **Continuous tilt tracking (beta.6, #33):** Tilt is recalculated and emitted
  every update cycle regardless of whether the resolved position changed.
- **Target Tilt sensor (beta.2, #33):** Diagnostic sensor exposing the engine's
  target tilt angle in degrees.

### 🐛 Bug Fixes

- **State-aware position settle — Shelly 2PM false-settle (beta.6, #33):** The
  settle poll was counting idle samples without checking whether the cover was
  still in motion. Three consecutive equal readings triggered "settled" while
  the motor was mid-travel, firing tilt prematurely. The stall counter now only
  increments when `cover.state` is not opening or closing.

- **Tilt drift stuck on stale target (beta.6, #33):** After a tilt command, the
  actual tilt reported by the cover could differ from the commanded angle by more
  than the hardware tolerance. Without the readback check, the coordinator treated
  the stale target as satisfied and stopped retrying. The fix clears the target
  when drift exceeds `VENETIAN_TILT_VERIFY_TOLERANCE` (5%), forcing a retry on
  the next cycle.

- **Boundary drift false override (beta.5, #33):** Drift exactly equal to the
  position tolerance floor passed the strict `<` check and reached the
  manual-override handler. Changing to `<=` closes the gap.

- **Two cooperating manual override bugs (beta.4, #33):** `SecondaryAxisCheck.evaluate`
  short-circuited to `consumed=False` when `new_value == expected`, skipping the
  suppression callback and letting position-axis drift reach the manual-override
  handler. Separately, `_rebase_commanded_position` captured `current_position`
  before the motor's back-drive settled, so the rebase was skipped and the
  commanded target drifted from the actual settled position. The fix checks
  suppression before the on-target early exit and adds a 1.5-second settle delay
  before the rebase.

- **False manual override from back-rotate drift (beta.3, #33):** When the
  back-rotate settled after the 90 s suppression window, `SecondaryAxisCheck`
  was letting the event propagate unconsumed into the manual-override handler.
  Setting `consumed=True` on the suppression-rejection path closes that gap.

- **Post-tilt reconciliation loop (beta.2, #33):** After `set_cover_tilt_position`
  fired, each subsequent `current_tilt_position` change re-triggered a tilt
  command. The reconciliation loop is now in-flight-aware and skips re-sends
  while the motor is settling.
