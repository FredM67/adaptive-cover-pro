# v2.21.2

## 🔑 Highlights

v2.21.2 closes the final reported edge cases in venetian tilt/move sequencing, then ships two features that make the integration significantly more useful in dashboards and diagnostics. Venetian users on KNX and Shelly hardware no longer see the "slats close then open" flicker on opening transitions or chronic position/tilt state divergence when the sun leaves the window's field of view. Dashboard authors can now opt in to a proxy cover entity per physical cover that enforces min-mode floors through native slider interactions — no YAML template workaround required. And the Decision Trace sensor now tells you exactly which weather condition is holding a cover closed, and whether it's still being held by the post-clearance timer after the condition has lifted.

## ✨ Features

- **Opt-in proxy cover for floor-aware dashboard sliders (#374, PR #379):** A new `enable_proxy_cover` toggle on the Covers & Device configuration step creates one `AdaptiveProxyCover` entity per physical cover. The proxy mirrors source cover state verbatim and intercepts `cover.set_cover_position`, `open_cover`, `close_cover`, and `set_cover_tilt_position` calls, routing them through a new shared `Coordinator.async_apply_user_position` helper that applies the same min-mode floor clamp already used by the `adaptive_cover_pro.set_position` service. Dashboard sliders — HA's built-in cover card, mushroom-cover-card, and others — now respect the floor without any custom YAML. `stop_cover` passes through unclamped; commands while the source cover is unavailable are silently dropped. DE/FR translations updated for the new toggle label and description.

- **Weather conditions surfaced on Decision Trace sensor (PR #380):** When the weather handler is controlling a cover (`control_method == ControlMethod.WEATHER`), the Decision Trace sensor now includes two additional attributes: `weather_active_conditions` (a list of currently-active condition labels — `wind_speed`, `rain_rate`, `is_raining`, `is_windy`, `severe_weather`) and `weather_in_clear_delay` (true while the post-clearance hold timer is still running). Both keys are absent when any other handler is in control, keeping the attribute namespace clean. `WeatherManager` gains `active_conditions` and `in_clear_delay` properties that back these attributes.

- **Tilt-first on venetian opening transitions (#33, PR #378):** KNX and Shelly venetian actuators briefly reassert their cached tilt against partially-closed slats during open travel, producing a visible "slats close then open" flicker. A new `CoverTypePolicy.before_position_command` hook (default no-op) is invoked by `CoverCommandService` before `set_cover_position` fires. `VenetianPolicy.before_position_command` implements the hook: on opening transitions (target > current position), it sends the new tilt via `_send_tilt_command(force=True, verify=False)` before the carriage moves. Closing transitions keep the existing position-then-tilt order — slats must close after the carriage has finished travelling. A target-unchanged dedup at the top of `_send_tilt_command` short-circuits the post-settle tilt resend in `run_sequence` when nothing changed, keeping total service-call count for an opening transition at 2.

## 🐛 Bug Fixes

- **Tilt verification falsely declares drift on lagging actuators (#33, PR #378):** Before this fix, `_verify_and_record_tilt` read `current_tilt_position` exactly 1.5 s after a tilt command and immediately declared drift if the reading didn't match. KNX and Shelly venetian actuators can publish their post-tilt state update 1–3 s after that window, so the verify routinely read the pre-update value, falsely flagged drift, cleared recorded tilt targets, and re-armed `maybe_update_tilt_only` to fire the same tilt command again next cycle — sometimes twice. `_verify_and_record_tilt` now polls up to `VENETIAN_TILT_VERIFY_MAX_SAMPLES` (4) times, `VENETIAN_TILT_VERIFY_POLL_SECONDS` (1.0 s) apart, accepting on the first in-tolerance sample. `tilt_command_drift` is emitted and the recorded target cleared only when every sample is out of tolerance. Healthy hardware still completes in roughly 1.5 s; lagging actuators get up to approximately 4.5 s to publish before drift is declared.

- **Suppressed venetian cycle leaves `_last_tilt` armed, replaying stale tilt (#33, PR #378):** When the cover's control method was non-SOLAR, or when SOLAR was active but direct sun was not striking the window, `post_pipeline_resolve`'s suppressed branch returned early without resetting `_last_tilt`. The prior solar tilt therefore kept replaying through `maybe_update_tilt_only` on every subsequent cycle, producing chronic position/tilt state divergence — the blind sat physically at 100/100 while HA continued reporting 100/55. A new `_clear_last_tilt()` method is called before the early-return so the tilt-only refresh stops at the correct time.

## 🧪 Testing

- 3169 tests passing.
- New tests in `tests/test_cover_types/test_venetian.py` cover the `before_position_command` hook: opening vs. closing transition branching, `None` tilt target short-circuit, and interaction with `_resolve_skip_above_tilt`.
- New tests in `tests/test_managers/test_dual_axis_sequencer.py` cover poll-retry behavior: immediate acceptance on the first in-tolerance sample, acceptance on a later sample after initial lag, and drift declaration only after all samples are out of tolerance.
- New tests in `tests/test_cover_types/test_venetian_post_pipeline.py` cover the `_clear_last_tilt` path: non-SOLAR control method clears a primed `_last_tilt`, SOLAR with `direct_sun_valid=False` clears it, and the `result is None` early-return leaves it untouched.
- New tests in `tests/cover/` cover proxy cover setup, state mirroring, command routing, min-mode floor clamping, availability, and reload lifecycle.
- New tests in `tests/test_decision_trace_enabled_handlers.py` and `tests/test_managers/test_weather.py` cover `WeatherManager.active_conditions`, `in_clear_delay`, and the Decision Trace sensor attributes.
